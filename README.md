# HH.ru Incremental ETL Project

## Описание

Проект реализует инкрементальный ETL для загрузки вакансий с HH.ru в PostgreSQL и в файловую систему в формате Parquet со сжатием Snappy.

## Что реализовано

- базовый ETL с инкрементальной загрузкой;
- логирование;
- обработка ошибок и ретраи;
- конфигурация отдельно от кода;
- генерация уникального `etl_id` и прокидывание его в данные и логи;
- выгрузка в файловую систему в колоночном формате `Parquet` со сжатием `Snappy`.

## Как работает загрузка

1. Из БД читается `High Water Mark` по полю `published_at`.
2. Из HH API запрашиваются вакансии новее этого значения.
3. Данные нормализуются в табличный вид.
4. Новые строки пишутся в PostgreSQL.
5. Этот же инкремент дополнительно сохраняется в `data/*.parquet`.

## Запуск

1. Установить зависимости:
   `pip install -r requirements.txt`
2. Поднять PostgreSQL.
3. Проверить `database.url` в `config.yaml`.
4. Открыть notebook и выполнить ячейки сверху вниз.
5. Запустить ячейку:
   `result_df = run_incremental_etl()`
